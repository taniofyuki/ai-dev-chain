# 設定・拡張ポイント設計

---

## 1. 設定のスコープ

### 1.1 プロジェクト単位設定

保存場所：`.ai-dev/project.yaml`

- フェーズ実行モード  
  - `manual`（手動）  
  - `continuous`（連続実行）※連続実行時は自動マージON
- 使用ロール構成  
  - `reviewer: true / false`
- 使用テンプレセット
- 整合性チェックルールの有効 / 無効
- 通知先  
  - GitHub Issue のみ

### 1.2 ユーザー単位設定

保存場所：`~/.ai-dev/config.yaml`

- GitHubアクセストークン
- デフォルトリポジトリ

---

## 2. 設定ファイル構成

| 種類 | 保存場所 | 形式 |
|-----|---------|------|
プロジェクト設定 | リポジトリ | `.ai-dev/project.yaml` |
ユーザー設定 | ローカル | `~/.ai-dev/config.yaml` |

---

## 3. 拡張ポイント

### 3.1 ロール拡張
- 新しいAIロールの追加  
  例：Security Reviewer / Performance Advisor

### 3.2 テンプレ拡張
- プロジェクト専用テンプレ
- フェーズ別テンプレの差し替え

### 3.3 チェックルール拡張
- 整合性チェックルールの追加・無効化
