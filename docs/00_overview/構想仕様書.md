# AI開発自動化システム 構想仕様書

## 1. 目的
本システムは、アプリケーション開発を定義された開発フローに従って実行し、  
各フェーズの成果物作成・確認・更新を AI により支援・自動化することで、  
開発を手順化された作業の流れとして安定的に回せるようにすることを目的とする。

---

## 2. 何をするシステムか
本システムは以下を行う。

- 開発フローをフェーズ単位で管理する
- 各フェーズの成果物を AI が生成する
- 成果物同士の整合性をチェックする
- GitHub 上に成果物を反映する
- 人が確認・承認して次フェーズへ進める

---

## 3. 開発フロー

| No | フェーズ | 実施事項 | 主な成果物 |
|---:|---|---|---|
| 1 | プロジェクト立ち上げ | 目的・スコープ定義 / 体制整理 / GitHubリポジトリ作成 / 基本ルール整備 | リポジトリ, 管理ページ |
| 2 | 要求整理 | 課題整理 / ゴール定義 / ユーザーストーリー作成 / 要件定義 | 要求仕様 |
| 3 | 方式選定 | 全体方針決定 / 構成要素整理 | 方針レポート |
| 4 | システム設計 | アーキテクチャ設計 / フロー整理 | システム設計書 |
| 5 | 技術スタック選定 | 言語・FW・DB・認証・ホスティング選定 | 技術選定レポート |
| 6 | 開発環境設計 | リポジトリ構成 / 設定管理 / 規約 / CI設計 | 環境設計書 |
| 7 | 詳細設計 | 画面設計 / API設計 / DB設計 | 詳細設計書 |
| 8 | 環境構築 | 雛形生成 / CI設定 / 起動導線 | 開発環境 |
| 9 | 実装 | FE/BE開発 / レビュー | ソースコード |
| 10 | テスト | 単体 / 結合 / 受入 | テストレポート |
| 11 | リリース | ビルド / リリースノート | アプリケーション |

※ テストはフェーズとして常に存在するが、実行は任意（スキップ可能）とする。

---

## 4. システムの動き

1. ユーザーがフェーズを開始  
2. AI が前フェーズ成果物を読み取り、次の成果物を生成  
3. 生成物をリポジトリへ反映  
4. 人が内容を確認・承認  
5. 承認後、次フェーズへ進む  

この流れを最終フェーズまで繰り返す。

---

## 5. 成果物の扱い
- 各フェーズの成果物は Markdown、設定ファイル、ソースコードとして管理する
- 成果物は常に GitHub リポジトリに保存され、履歴管理される
- 後続フェーズは常に最新の成果物を入力として使用する

---

## 6. 人とAIの役割分担

| 役割 | 内容 |
|---|---|
AI | 成果物の生成、修正案作成、整合性チェック |
人 | 意思決定、最終承認、方針変更 |

---

## 7. ゴール

- 開発手順が明確で迷わない  
- 成果物が常に揃っている  
- 途中からでも状態が把握できる  
- 開発の再現性が高い
